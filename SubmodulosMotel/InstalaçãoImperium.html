1. Instalação do Imperium

Primeiro passo: instalar o sistema Imperium no servidor.

Procedimento de Instalação
  • Execute o instalador do Imperium.
  • Clique apenas em Avançar em todas as telas.
  • Verifique se a opção "Criar atalho na área de trabalho" está marcada.
  • Se a opção estiver desmarcada, marque-a antes de prosseguir.

    • 2. Cópia de Arquivos Essenciais

    Copiar arquivos essenciais para locais específicos do sistema.

    Arquivos a Serem Copiados
  • Copie criartablespace.sql de C:\Bitz Softwares\Utilitários para a raiz: C:\
  • Copie todos os arquivos de C:\Bitz Softwares\App\Dll e cole em: C:\Bitz Softwares\App\
  • Extraia os arquivos de Dlls.rar na pasta: C:\Bitz Softwares\App\
  • Crie a pasta Atualizadores em: C:\Bitz Softwares\

    • 3. Instalação do Oracle

    Instalar o banco de dados Oracle XE necessário para o sistema.

    Procedimento de Instalação
  • Inicie o instalador do Oracle.
  • Clique somente em Avançar em todas as telas.
  • Na aba de Senha, digite: bitzdba
  • Aguarde a finalização completa da instalação.
    • ATENÇÃO

    Certifique-se de usar a senha bitzdba durante a instalação do Oracle.

    4. Instalação do Firebird

    Instalar o Firebird para funcionamento de alguns módulos do sistema.

    Procedimento de Instalação
  • Inicie o instalador do Firebird, localizado em C:\Bitz Softwares\Utilitários.
  • Clique em Avançar em todas as telas.
  • Aguarde a conclusão da instalação.

    • 5. Comandos SQL - Configuração Tablespace

    Executar comandos SQL para configurar o tablespace e permissões do Oracle.

    Execução de Comandos SQL
  • Abra o programa "Executar linhas de comando SQL".
  • Digite os comandos na sequência, linha por linha:
    • Comandos SQL a serem executados:
    connect system
    # Senha: bitzdba
    bitzdba
    @c:/criartablespace
    disconnect
    connect /as sysdba
    grant create any directory to BITZDBA;
    create directory EXP_DIR as 'X:\Backup';
    exit
    Nota: Se for necessário alterar o diretório do Backup:
    drop directory EXP_DIR;
    create directory EXP_DIR as 'X:\Backup';

    6. Atualização e Preparação dos Arquivos

    Localizar e copiar os arquivos mais recentes do sistema.

    Localização dos Arquivos
  • Busque a versão mais recente em T:\VERSOES\ano\mês.
  • Arquivos Principais a procurar:
    • BDAT.FDB
    • imperium.exe
    • ServAutomacao.exe
    • Outros executáveis específicos (Auto-Atendimento, GateCam)
  • Sincronização: Todos os arquivos, exceto BITZSincronizador.exe, estão em: T:\IMPLANTACAO\Instaladores\Instalação Sinc
  • O executável BITZSincronizador.exe está em T:\VERSOES\ano\mês
  • Transfira todos os arquivos para: C:\Bitz Softwares\Atualizadores
    • Substituição dos Arquivos Antigos
  • Vá para C:\Bitz Softwares\App\ e apague os executáveis antigos.
  • Vá para C:\Bitz Softwares\BD\ e apague o arquivo BDAT.FDB.
  • Copie os novos arquivos da pasta Atualizadores:
    • Arquivos .EXE para: C:\Bitz Softwares\App\
    • Arquivo BDAT.FDB para: C:\Bitz Softwares\BD\
  • Transfira a base de dados do cliente (arquivo .DMP) para: X:\Backup

    • 7. Configuração do Agendamento de Backup

    Configurar o aplicativo BitzBackup para backups automáticos.

    Configuração do BitzBackup
  • Execute BitzBackup.exe em C:\Bitz Softwares\App\
  • O aplicativo inicia minimizado - clique duas vezes no ícone na barra de tarefas.
  • Na tela Backup, preencha os campos:
    • Pasta BIN Oracle: C:\oraclexe\app\oracle\product\10.2.0\server\BIN
    • Pasta de Destino Backup: X:\Backup
    • Pasta de Backup Remoto: C:\Program Files (x86)\Google\Drive
    • Pasta do WinRar: C:\Program Files\WinRAR
  • Marque a programação: TODOS os dias e a cada 4 horas
  • Clique em Salvar
  • Crie um atalho do BitzBackup.exe e cole-o na pasta Inicializar do Windows
    • ATENÇÃO

    Instale o WinRar se necessário (instalador disponível em C:\Bitz Softwares\Utilitários)

    8. Restauração da Base de Dados do Cliente

    Restaurar a base de dados do cliente usando o BitzBackup.

    Procedimento de Restauração
  • Na aba Restore do BitzBackup, em "Arquivo a Restaurar", selecione o arquivo .DMP do cliente que está em X:\Backup
  • Clique em Restaurar Backup
  • O aplicativo será minimizado (ícone de computador na barra de tarefas)
  • Aguarde até que:
    • O ícone volte a ser o ícone padrão da Bitz
    • Uma notificação informe que o processo foi finalizado

    • 9. Finalização e Liberação de Porta

    Finalizar a instalação e configurar acessos de rede.

    Atualização da Base de Dados
  • Abra o aplicativo BITZUpdateDB.exe em C:\Bitz Softwares\Utilitários
  • Em "Banco de Dados Atualizador", clique na lupa e selecione o arquivo BDAT.FDB em C:\Bitz Softwares\BD\
  • Corrija o caminho para: 127.0.0.1:C:\BITZ Softwares\BD\BDAT.FDB
  • Clique em Atualizar e aguarde a mensagem de sucesso
    • Liberação da Porta 1521
  • Libere a porta 1521 no firewall para o IP do servidor
  • Próximo Passo: Iniciar os treinamentos de uso do sistema com o cliente

    • 10. Configuração de Serviços Externos

    Configurar No-IP e Google Drive para acesso remoto e backups.

    Instalação e Cadastro do No-IP
  • Inicie a instalação do DUC (disponível em C:\Bitz Softwares\Utilitários)
  • Acesse o site No-IP na sua máquina, não na do cliente:
    Acesso No-IP:
    Login: bitz@bitzsoftwares.com.br
    Senha: bitz2012
  • Clique em Add a Host
  • Crie o No-IP com o nome do motel (ex: statusmotelvt)
  • Após criar, volte à máquina do cliente e vincule o No-IP no aplicativo DUC
  • IMPORTANTE: Em Edit Host, marque: Require a password to modify hosts
    • Instalação do Google Drive
  • Instale o Google Drive (instalador em C:\Bitz Softwares\Utilitários)
  • Na sua máquina, crie uma nova conta Google para o motel (ex: statusmotelvt@gmail.com)
  • Na máquina do cliente, faça login no Google Drive com a conta recém-criada
    • Pasta de Backup Local
  • Crie uma pasta chamada Backup na raiz:
  • Prioridade: Disco D:\Backup
  • Alternativa: Se houver apenas uma partição, crie em C:\Backup
    • Instalação Concluída! O Sistema Imperium foi instalado e configurado com sucesso. O sistema está pronto para uso.
